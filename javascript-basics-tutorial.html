JavaScript Basics Tutorial

After you finished learning HTML and CSS it's useful to make your page more interactive. In this point you encounter the king of front-end websites - JavaScript. For those who did something in C++ or C# it won't be hard - JS is weird, but while learning it, you will discover that JS is more simple than these.

How to attach scripts to your page

To use JS on your website - you need to add script mark to your website
For training, in your folder create index.html file: 

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My first JavaScript code!</title>
</head>
<body>
    
</body>
</html>

Now we shoud add <script> to your file. You can add it to <head> or end of <body>

<!DOCTYPE html>
    <html lang="en">
    <head>
        <meta charset="UTF-8">
        <meta name="viewport" content="width=device-width, initial-scale=1.0">
        <title>My first JavaScript code!</title>

    </head>
    <body>
        <!-- Your html -->

    </body>
</html>

Your code can be placed in many ways. You can add it in index.html like that:

<!DOCTYPE html>
    <html lang="en">
    <head>
        <meta charset="UTF-8">

Or you can add it in seperate file by adding src atrribute to <script>: 

<!DOCTYPE html>
    <html lang="en">

        <meta name="viewport" content="width=device-width, initial-scale=1.0">
        <title>My first JavaScript code!</title>

    </body>

And your script.js file:

// Your code in seperate file

Which should you use? That depends. If you want to use your script in many pages, then you shoud embed your code in seperate file (you can also copy your code to all of your pages, but it's just a waste of your place on your server). If you want your page to load quickly - you shoud add your code inside index.html - every time your page loads a new file it takes some time.

"Async" and "defer" attributes

You can add two special attributes to script tag - "async" and "defer".

<script async></script>
<script defer></script>
<script async defer></script>

"Async" attribute make your script being loaded during rest of your html being loaded. When your script finish loading, your code will be executed.

"Defer" attribute make your script being loaded during rest of your html being loaded. When your page finish loading, your code will be executed (precisely it will be executed just before DOMContentLoaded event).

Important note!


When you decide to add your code in head part of your page, give your script tag "defer" attribute. Otherwise you will get loads of errors instead of results you wanted.

<!DOCTYPE html>
    <html lang="en">
    <head>
        <meta charset="UTF-8">
        <meta name="viewport" content="width=device-width, initial-scale=1.0">
        <title>My first JavaScript code!</title>
        <script defer></script> <!-- No errors :) -->
    </head>
    <body>
        <!-- Your html -->
    </body>
</html>

Script type

Script tag also can have type attribute. If we are talking about JavaScript, it has two possible values: 

<script type="text/javascript"></script>
<script type="module"></script>

The first value is practically dying, because browsers set this automatically (if it's not set manually of course). The second option is used if we use modules (more details about it will be on my website in the future).

Dialog boxes

The are three types of dialog boxes in JS: 

alert("Your message"); // Displays alert box with a message
confirm("Delete Windows folder"); // Displays a dialog box with two buttons - OK and Cancel. It returns true or false
prompt("Your secret password","Input field default value"); // Displays a dialog box with input text field. Returns this field value

You should use them if you want to communicate with the user. When you are little more advanced in JS you should stop use it and make your own pop-up window. JS built-in dialog boxes cannot be customized, they often don't fit to your website look. They are simple to use, but they don't look good.

Variables and operators

While your code is running at some point you will need to store some values to use them later. That's what variables do. There are two ways to create a variable:

var myVariable1 = "some text"; // Old method
let myVariable2 = "some other text"; // New method

They are both good. The main differece between them is the range - "var" defined in a function cannot be used outside it, "let" defined in a block of code cannot be used outside it (you will understand it later):

// function
function myFunction(){
    var myVariable = "Hello World";
    alert(myVariable); // OK
}
alert(myVariable); // Throws an error

// block
{
    let myVariable = "Hello World";
    alert(myVariable); // OK
}
alert(myVariable); // Throws an error

You can store many types of data in a variable:

let num = 14000605; // A number
let logic = true; // A boolean value - true/false
let textString = "Hello world"; //A string - text value
let undef = undefined; // undefined - displays when you use undefined variable or use a variable that you don;t have access to
let nullValue = null; // null - nothing - being honest I don't know why it exist
let list = [1,2,3,4] // array - a list that contains many variables of many types - more datails later
let object = {value1:"value"} // object - more details later

There are strict rules how to name your variables:

// OK:
let myAge;

let $my_age;

// Thorws an error:

Operators

Operators are math in programming. All of the operators are presented in code below: 

let x = 5;
let y = 2;
let z = 3

// Basic math operators
alert(x + y); // 5 + 2 = 7 
alert(x - y); // 5 - 2 = 3
alert(x * y); // 5 * 2 = 10
alert(x / y); // 5 / 2 = 2.5
alert(x % y); // 5 % 2 = 1, modulo, rest of divison
alert(x ** y); // 5 ** 2 = 25, 5 to the power of 2

z-- // same as z = z - 1
z++ // same as z = z + 1

// Assignment operators
x = y // x = 2
x += y // same as x = x + y
x -= y // same as x = x - y
x *= y // same as x = x * y
x /= y // same as x = x / y
x %= y // same as x = x % y

// Comparison operators
let a = 5
let b = 7

alert(a == b); // false, a equal to b
alert(a != b); // true, a not equal to b
alert(a === b); // false, a equal to b and the same data type
alert(a !== b); // true, a not equal to b or different data type
alert(a > b); // false, a is more than b
alert(a < b); // true, a is less than b
alert(a >= b); // false, a is more than or equal b
alert(a <= b); // true, a is less than or equal b

// Logic operators
alert((a != b) && (a == b)); // false, AND operator
alert((a != b) || (a == b)); // true, OR operator
alert((a != b) ^ (a < b)); // false, XOR operator
alert(!(a < b)); // false, NOT operator

!false // it's funny because it's true

Conditional statements

Conditional statements are used when you need to check if statement is true.

if(statement){
    // do sth if statement == true
}
else if(otherStatement){
    // do sth if otherStatement == true and statement == false
}
// in theory you can add infinite else if statements
else{
    // do sth if statement == false and otherStatement == false
}

Note that statement doesn't need to be a boolean - if we type sth that is not a boolean, JS will convert it to boolean. There are some values which are always converted to false:

Boolean(false); //false
Boolean(null); //false
Boolean(undefined); //false
Boolean(0); //false
Boolean(NaN); //false
Boolean(""); //false
Boolean(document.all); //false

Other values will be always converted to true.

There is sth called ternary operator. It's shortened version of if/else statement:

let variable = (statement) ? if_true : if_false;

let i;
if(true){
    i = 10;
}
else{
    i = 5;
}

// The same using ternary operator
let i = (true) ? 10 : 5;


Other method to create conditions is switch. It allows you to check if variable equal something:

let variable = prompt("Enter code:");
switch(variable){
    case 0:
    // some code
    break;

    case 1:
    // some other code
    break;

    default:
    // other values than 0 and 1
}

Loops

If you want to do sth mutliple times, you should use a loop. There are a few times of loops. First is "for" loop. It executes code a specific number of times:

for(variable_iniciation; statement_that_ends_the_loop; variable_change){
    // do sth  
}

// loop from 0 to 99, executes 100 times
for(let i = 0; i < 100; i++){
    // do sth  
}

// loop form 100 to 1, also executes 100 times
for(let i = 100; i > 0; i--){
    // do sth  
}

// infinite loop - not recommended - it's a mass of lags
for(;;){
    // do sth infinite times
}

"For" loop can also bo used with "of" and "in" keywords. The first one allows you to iterate in arrays, second to iterate in objects (better explanations in the future):

for(let user of users){
    // do sth with user
}

for(let data in object){
    // do sth with data
}

There are also loops that execute code only if some statement is true - "while" and "do-while" loops. The main difference between them is that "do-while" loop will be executed at least one time:

while(statement){
    // do sth or do nothing
}

do{
    //do sth at least one time
}while(statement);

You can do loops in a loop:

for(let x = 0; x < 100; x++){
    for(let y = 0; y < 100; y++){
        // do sth multidimesional
    }
}

If you want the loop to stop executing code, you should use break method. If you want the loop to skip loop iteration, you should use continue method.

let is_user_active = false;
for(let user of active_users){
    if(user == "TheShaman"){
        is_user_active = true;
        break; // you don't need to search anymore
    }
}

for(let user of users){
    if(user == "noobmaster69"){
        continue; // noobmaster69 has been banned - continue
    }
    // do sth with unbanned users
}
